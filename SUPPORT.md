# Support

Thanks for using Whity Core! Here's where to go depending on what you need.

## Bug reports & feature requests

Use [GitHub Issues](https://github.com/AmroKSaleh/whity-core/issues) —
please search first to avoid duplicates. A good bug report includes:

- What you expected to happen vs. what happened
- Steps to reproduce (commands run, env vars, minimal config)
- Whity Core version / commit, PHP version, deployment mode (Docker Compose,
  bare-metal, etc.)
- Relevant logs (redact secrets and tenant data)

## Security vulnerabilities

**Do not open a public issue.** See [SECURITY.md](SECURITY.md) for the
private reporting process.

## Questions & general discussion

Open a [GitHub Discussion](https://github.com/AmroKSaleh/whity-core/discussions)
if enabled on the repository, or open an issue tagged `question` if not.

## Documentation

Start with [`docs/wiki/`](docs/wiki/) — architecture, deployment, and
developer guides live there. The rendered wiki mirrors this directory (see
`.github/workflows/sync-wiki.yml`).

## Commercial support

There is no paid support tier today. If you need dedicated help (custom
deployment, plugin development, integration work), reach out to
**amroksaleh@gmail.com**.
